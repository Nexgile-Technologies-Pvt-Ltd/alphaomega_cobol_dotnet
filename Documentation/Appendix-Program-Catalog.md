## Catalog

This catalog is the path-level workload index for the supplied source snapshot. It is derived from the filesystem inventory, then reconciled with the executable behavior documented in [Online screens and navigation](04-Online-Screens-and-Navigation.md#transaction-and-screen-catalog), [Batch processing](05-Batch-Processing.md#batch-workload-catalog), and [Optional modules and integrations](07-Optional-Modules-and-Integrations.md#module-boundaries).

| Artifact class | Supplied paths | Reconciliation |
|---|---:|---|
| COBOL | 44 | 17 core online, 14 core batch/utility, 3 optional transaction-type, 8 optional authorization, 2 optional VSAM/MQ |
| Assembler and macros | 4 | 2 assembler sources and 2 macros |
| JCL | 55 | 38 core application, 8 optional-module, 9 sample build/platform |
| Cataloged procedures | 6 | 2 application procedures and 4 sample build procedures |
| **Catalog total** | **109** | Every path is present in the [source inventory](Appendix-Source-Inventory.md#inventory-snapshot) |

The Purpose, Invocation/type, and Key I/O/dependencies columns are **Observed — code**. The Target mapping/disposition column is a **Target recommendation**. A defect link identifies material behavior that must not be silently copied; a dash means this appendix adds no separate defect claim beyond the cited functional page.

Path spelling and case match the repository. Forward slashes are used deliberately so an automated coverage check can compare this document with the filesystem.
